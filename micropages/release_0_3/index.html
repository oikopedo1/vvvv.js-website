Granted, this version's micropage does not show off visually as much as the previous one, but this has not been the focus of this version update anyway. Instead, version 0.3.0 comes with improvements under VVVV.js' hood — improvements in performance, stability and functionality which bring VVVV.js a step closer to becoming a ready-for-production development tool. Here's what's new:

Subpatches

The biggest addition in this version is the support for subpatches. If you are into VVVV or similar software, you already know how valuable these are. For those who are not familiar with the subpatch concept: you can now group together a patch and form a 'subpatch', which you can use as a node in another patch. You can think of it as functions or classes/modules, if you like. This is great for modularizing your patches, hiding complexity and increasing reusability of your code. Read more about subpatches in the VVVV Documentation.

The Clock Pattern Example

If everything works as supposed, you should see a pattern consisting of repeating clock designs at the top of this page. You can observe the according patch on the right. The patch uses subpatches to modularize the different clock designs, and leaves only switching between them to the main page.
